# 二手物品交易管理系统项目文档

## 第一部分  技术栈和工具

1.  后端： **JDK 8+**、**Maven 3.6+**、**Spring Boot 2.7+**、**Mybatis 2.3.0**、**Minio 8.2+**、**Springdoc 1.6.14**；
2.  前端：**Node.js 16+**、**vue-admin-template 4.4.0**
3.  数据库：**MySQL 8.0**
4.  安装开发工具：**IDEA（后端）**、**WebStorm（前端）**、**Apifox和Swagger UI（接口调试）**
5.  版本管理：**Git**

## 第二部分 后端

> GitHub项目地址：https://github.com/Myj-2024/second-hand-trading-system

### 说明

所有接口基础请求地址：`http://localhost:8080`。
统一响应格式核心字段：`code`(响应码)、`msg`(提示信息)、`data`(返回数据)。
请求参数中标注**必须**的字段为接口调用必填项，**非必须**字段可根据业务需求选择性传入。

### 一、文件上传模块

#### 1. 文件上传接口

##### 基本信息

请求路径：`/upload`
请求方式：`POST`
接口描述：用于上传商品图片、用户头像等文件，返回文件访问路径

##### 请求参数

参数格式：`multipart/form-data`
参数说明：

| 名称 | 类型   | 是否必须 | 备注                      |
| ---- | ------ | -------- | ------------------------- |
| file | binary | 必须     | 待上传的文件（图片/文档） |

##### 响应数据

参数格式：`application/json`
参数说明：

| 名称 | 类型    | 是否必须 | 备注                    |
| ---- | ------- | -------- | ----------------------- |
| code | integer | 必须     | 响应码，成功返回200     |
| msg  | string  | 非必须   | 操作提示信息            |
| data | string  | 必须     | 上传成功后的文件访问URL |

响应参数示例：

```json
{
    "code": 200,
    "msg": "文件上传成功",
    "data": "http://localhost:8080/upload/images/20260131/avatar_123.jpg"
}
```

### 二、用户管理模块

#### 1. 用户分页查询

##### 基本信息

请求路径：`/admin/user/page`
请求方式：`GET`
接口描述：根据用户名、状态等条件分页查询后台用户列表

##### 请求参数

参数格式：`query`（URL查询参数）
参数说明：

| 名称     | 类型    | 是否必须 | 备注                  |
| -------- | ------- | -------- | --------------------- |
| username | string  | 非必须   | 用户名（模糊查询）    |
| pageNum  | integer | 必须     | 页码，从1开始         |
| pageSize | integer | 必须     | 每页显示条数          |
| status   | integer | 非必须   | 用户状态（启用/禁用） |

请求参数示例：

```
pageNum=1&pageSize=10&username=admin&status=1
```

##### 响应数据

参数格式：`application/json`
参数说明：

| 名称               | 类型    | 是否必须 | 备注                   |
| ------------------ | ------- | -------- | ---------------------- |
| code               | integer | 必须     | 响应码，成功返回200    |
| msg                | string  | 非必须   | 操作提示信息           |
| data               | object  | 必须     | 分页返回数据           |
| \|- total          | long    | 必须     | 符合条件的用户总记录数 |
| \|- records        | array   | 必须     | 当期页用户列表         |
| \|- \|- id         | long    | 必须     | 用户ID                 |
| \|- \|- username   | string  | 必须     | 登录用户名             |
| \|- \|- nickname   | string  | 非必须   | 用户昵称               |
| \|- \|- phone      | string  | 非必须   | 联系电话               |
| \|- \|- avatar     | string  | 非必须   | 用户头像URL            |
| \|- \|- status     | integer | 必须     | 用户状态               |
| \|- \|- createTime | string  | 非必须   | 用户创建时间           |

响应参数示例：

```json
{
    "code": 200,
    "msg": "查询成功",
    "data": {
        "total": 2,
        "records": [
            {
                "id": 1,
                "username": "admin",
                "nickname": "系统管理员",
                "phone": "13800138000",
                "avatar": "http://localhost:8080/upload/avatar/admin.jpg",
                "status": 1,
                "createTime": "2026-01-01T00:00:00"
            },
            {
                "id": 2,
                "username": "editor",
                "nickname": "内容编辑",
                "phone": "13900139000",
                "avatar": null,
                "status": 1,
                "createTime": "2026-01-02T10:00:00"
            }
        ]
    }
}
```

### 说明

前端使用[`vue-admin-template`](https://github.com/PanJiaChen/vue-admin-template)模板，全程使用**豆包AI**辅助修改和编写相关代码。具体操作如下：

1.  下载本项目前端GitHub仓库，执行命令克隆模板：

```bash
    git clone https://github.com/Myj-2024/second-hand-trading-system-ui.git
```

2.  进入`second-hand-trading-system-ui`目录，安装依赖：`npm install`
3.  启动测试：`npm run dev`，访问 `http://localhost:9528` 即可登录页面。
4.  修改接口基础地址，打开 `src/utils/request.js`，修改 `baseURL` 为后端接口地址：

```javascript
    const service = axios.create({
      baseURL: 'http://localhost:8080', // 后端项目端口
      timeout: 5000
    })
```
