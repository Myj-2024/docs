# 中小微企业服务系统API文档
## 文档说明
本文档基于OpenAPI 3.1.0规范生成，涵盖中小微企业服务系统的所有核心接口，包含接口地址、请求方式、参数说明、响应格式等关键信息，适用于前后端开发对接和接口测试。

### 基础信息
| 项目 | 说明 |
|------|------|
| 文档版本 | 1.0.0 |
| 开发环境地址 | http://localhost:8080 |
| 接口前缀 | 所有接口均基于根路径，具体路径见各接口说明 |
| 通用响应格式 | 所有接口返回统一格式，包含code、message、data、timestamp字段 |

## 一、通用响应结构
### 1.1 基础响应对象

```json
{
  "code": 200,          // 状态码（200成功，其他为异常）
  "message": "操作成功", // 提示信息
  "data": {},           // 业务数据（不同接口返回不同结构）
  "timestamp": 1740672000000 // 响应时间戳（毫秒）
}
```

### 1.2 常用响应类型说明
| 响应对象 | 说明 |
|---------|------|
| Result | 基础响应，data为任意对象 |
| ResultVoid | 无业务数据响应（data为空） |
| ResultString | data为字符串类型（如Token、文件URL） |
| ResultPageResult | 分页响应，data包含total（总数）和records（列表） |
| ResultListXXX | 列表响应，data为指定类型的数组 |

| 接口名称 | 请求路径 | 请求方式 | 核心功能 |

#### 2.1.1 用户登录
**请求路径**：`/admin/auth/login`  
**请求方式**：POST  
**请求体**：
| 参数名 | 类型 | 是否必填 | 描述 |
|--------|------|----------|------|
| username | string | 是 | 用户账号 |
| password | string | 是 | 用户密码 |

**响应格式**（Result用户VO）：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1001,
    "username": "admin",
    "realName": "系统管理员",
    "token": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiJhZG1pbiJ9.xxx"

  "timestamp": 1740672000000
```

### 3.2 枚举说明
#### 3.2.1 包抓联问题状态
| 状态值 | 描述 |
|--------|------|
| 受理中 | 问题已接收，待处理 |
| 办理中 | 问题正在处理 |
| 办结中 | 问题处理完毕，待确认 |
| 完全办结 | 问题已彻底解决 |
| 暂无法办结 | 因特殊原因无法解决 |

| 状态码 | 描述 |
|--------|------|
| 200 | 操作成功 |
| 400 | 参数错误 |
| 401 | 未授权（Token失效/未登录） |
| 403 | 权限不足 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 四、接口调用规范
### 4.1 通用规则
1. **请求头**：所有非登录接口需在请求头携带`Authorization: Bearer {token}`
2. **字符编码**：统一使用UTF-8编码
3. **时间格式**：所有时间字段统一为`yyyy-MM-dd HH:mm:ss`格式
4. **分页参数**：分页接口统一使用pageNum（页码）和pageSize（每页条数），默认pageNum=1，pageSize=10

### 4.2 错误处理
1. 接口调用失败时，code返回非200值，message返回具体错误信息
2. 业务异常需返回明确的错误提示，如"用户名已存在"、"权限不足"等
3. 系统异常返回通用提示，如"服务器内部错误，请联系管理员"

2. 所有接口返回统一格式的响应体，包含状态码、提示信息、业务数据和时间戳，便于前端统一处理。
3. 核心业务模块包括用户权限、企业管理、包抓联服务、政策管理等，接口设计符合RESTful风格，支持分页、条件查询、增删改查等基础操作。
