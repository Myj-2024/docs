# 中小微企业服务平台项目文档
## 一、项目概述
### 1.1 项目背景
为优化营商环境，提升中小微企业服务效率，实现**企业管理、政策发布、包抓联服务、通知推送、权限管控**全流程数字化，特开发本平台，打通政府部门与企业间的服务壁垒。

### 1.2 项目目标
1. 构建统一的企业信息管理库，实现企业数据标准化、可追溯
2. 搭建政策通告、通知公告精准推送通道，提升信息触达率
3. 实现县级领导包抓联企业、问题上报-受理-办理-办结全流程闭环
4. 完善权限体系，支持多角色、多部门分级分权管理
5. 提供前台门户展示与后台管理双端能力

### 1.3 系统架构
- **前端**：Vue3 + Element Plus + Pinia + Vite
- **后端**：SpringBoot + MyBatis-Plus + MySQL + MinIO（文件存储）
- **接口规范**：OpenAPI 3.1.0（Swagger）
- **部署模式**：前后端分离，开发环境地址 `http://localhost:8080`

## 二、数据库设计
### 2.1 核心表结构说明
| 表名 | 模块 | 说明 |
|------|------|------|
| `sys_user` | 系统管理 | 系统用户（管理员/部门账号） |
| `sys_role` | 系统管理 | 角色定义（ADMIN/DEPT/USER） |
| `sys_permission` | 系统管理 | 菜单权限与路由配置 |
| `sys_dept` | 系统管理 | 政府部门/乡镇信息 |
| `sys_dict`/`sys_dict_item` | 系统管理 | 全局字典（企业类型、行业、状态等） |
| `sys_icon` | 系统管理 | 菜单/页面图标库 |
| `sme_enterprise` | 企业管理 | 中小微企业基础信息 |
| `sme_lpe` | 包抓联管理 | 县级领导包抓企业关系 |
| `sme_package_contact` | 包抓联管理 | 企业问题上报与办理主表 |
| `sme_package_handle_record` | 包抓联管理 | 问题办理进度与附件记录 |
| `sme_policy` | 政策管理 | 政策发布、置顶、展示控制 |
| `sme_notice` | 通知管理 | 通知公告（全员/指定用户） |
| `sme_notice_user` | 通知管理 | 通知-用户已读关联 |

### 2.2 表关系概要
1. **用户-角色-权限**：一对一角色，角色绑定菜单权限，实现动态路由
2. **企业-包抓联**：一个企业对应一个包抓领导、一个责任部门
3. **问题-办理记录**：一个问题可多条办理记录，支持附件上传
4. **通知-用户**：一条通知可推送给多用户，记录已读状态

## 三、接口文档说明
### 3.1 接口访问地址
- **Swagger 地址**：http://localhost:8080/swagger-ui/index.html
- **OpenAPI 规范**：OpenAPI 3.1.0
- **基础路径**：`http://localhost:8080`

### 3.2 核心接口分类
| 模块 | 接口数量 | 核心能力 |
|------|----------|----------|
| 用户登录 | 2 | 登录、刷新Token |
| 用户管理 | 多接口 | 增删改查、重置密码、修改状态 |
| 角色权限 | 多接口 | 角色管理、权限分配 |
| 企业管理 | 多接口 | 企业增删改查、状态控制 |
| 包抓联管理 | 多接口 | 包抓关系、问题办理、进度记录 |
| 政策管理 | 多接口 | 政策发布、置顶、隐藏、批量操作 |
| 通知管理 | 多接口 | 通知发送、已读标记、未读计数 |
| 字典/部门/图标 | 多接口 | 基础数据维护 |

### 3.3 接口通用规范
- **请求方式**：GET（查询）、POST（新增/分页）、PUT（修改）、DELETE（删除）
- **请求头**：登录接口需携带 `Authorization: Bearer {token}`
- **返回格式**

```json
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1734567890123
}
```

- **分页统一返回**：`PageResult`（total + records）

## 四、前端项目说明
### 4.1 技术栈
- 框架：Vue3 + Vite
- UI：Element Plus
- 状态管理：Pinia
- 路由：Vue Router
- 文件存储：MinIO

### 4.2 路由设计
#### 4.2.1 静态路由（门户 + 管理台入口）

```js
// 门户路由（/home、/policy、/guide、/enterprise、/contact）
// 管理台路由（/admin/dashboard）
```

#### 4.2.2 动态路由
- 来源：后端 `userStore.menus` 菜单树
- 自动挂载：路由守卫中加载并添加到 `Layout` 下
- 详情路由（隐藏）：字典项、通知详情、办理进度详情

#### 4.2.3 路由工具函数
- `resolveComponent`：自动匹配视图文件
- `transformRoute`：后端菜单转前端路由
- `addDynamicRoutes`：动态添加路由
- `resetRouter`：退出登录重置路由

### 4.3 路由守卫（权限控制）
#### 核心逻辑
1. **已登录**：访问 `/` 或 `/home` 自动跳 `/admin/dashboard`
2. **白名单放行**：`/home、/policy、/guide、/enterprise、/contact
3. **未登录**：访问 `/admin/*` 拦截并提示登录，跳转到首页
4. **已登录无菜单**：请求用户信息 → 加载动态路由 → 跳转目标页
5. **异常处理**：Token 失效自动登出

## 五、系统功能模块
### 5.1 前台门户（无需登录）
1. **首页**：平台概况、快捷入口
2. **政策通告**：展示已发布政策、通知
3. **办事指南**：企业办事流程说明
4. **企业风采**：展示合规企业信息
5. **联系我们**：政府部门联系方式

### 5.2 后台管理（需登录）
1. **控制台**：数据概览
2. **系统管理**
   - 用户管理：账号、角色、部门、状态
   - 角色管理：角色增删改、权限分配
   - 菜单管理：路由配置、图标、权限码
   - 字典管理：全局数据字典维护
   - 部门管理：政府部门/乡镇信息
   - 图标管理：菜单图标上传与管理
3. **企业管理**：企业信息录入、编辑、状态控制
4. **包抓联管理**
   - 包抓关系配置：领导-企业-部门绑定
   - 问题上报：企业问题录入
   - 问题办理：受理、办理、办结、无法办结、进度记录、附件上传
   - 办理进度详情：全流程时间线展示
5. **政策管理**：政策发布、编辑、置顶、显示/隐藏、批量操作
6. **通知管理**
   - 通知发布：指定类型、接收人（全员/指定用户）
   - 我的通知：未读计数、已读标记
   - 发送记录：本人发布的通知列表

## 六、权限体系设计
### 6.1 角色定义
1. **管理员（ADMIN）**：最高权限，全功能访问
2. **部门管理员（DEPT）**：业务功能（通知、政策、包抓联）增删改查
3. **普通用户（USER）**：仅查看自身权限数据

### 6.2 权限控制粒度
1. **菜单级**：控制左侧菜单显示/隐藏
2. **按钮级**：控制新增、编辑、删除、办理等操作
3. **路由级**：未授权路由自动拦截
4. **数据级**：部门账号仅操作本部门相关数据

### 6.3 动态路由实现
1. 登录 → 获取Token → 请求用户信息与菜单树
2. 菜单树 → 转换为前端路由 → 挂载到管理台Layout
3. 路由守卫校验权限，无权限自动拦截

## 八、项目规范
### 8.1 编码规范
1. 前端：Vue3 组合式API，命名采用驼峰，文件名为小写+横线
2. 后端：RESTful 接口，统一返回格式，字段下划线命名
3. 数据库：表名、字段下划线，注释完整，逻辑删除 `del_flag`

### 8.2 业务规范
1. 企业信息必须统一社会信用代码、法人、联系方式
2. 包抓联问题必须：受理 → 办理 → 办结/无法办结
3. 通知/政策支持：发布、编辑、删除、置顶、隐藏
4. 文件上传统一使用 MinIO，返回可访问URL

## 九、异常与安全
1. **Token 过期**：自动登出，跳转首页
2. **无权限访问**：路由守卫拦截，提示无权限
3. **接口异常**：统一错误码与提示信息
4. **密码安全**：BCrypt 加密存储
5. **文件安全**：仅允许上传指定格式，后台校验

## 十、文档总结
本项目为**中小微企业服务平台**，采用前后端分离架构，实现了**门户展示、企业管理、政策通知、包抓联闭环服务、精细化权限管理**五大核心能力。
- 前端：动态路由、权限守卫、统一布局
- 后端：标准接口、完整数据库、文件存储
- 业务：覆盖政府服务企业全流程，可直接上线使用
